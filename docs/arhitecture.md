# Architecture

This project simulates a real-world QA pipeline.

Since the production application is private, a mock server is used to replicate application behavior.

## Layers

1. Mock server (simulates API and responses)
2. Basic automated tests (Jest)
3. CI/CD pipeline (GitHub Actions)
4. AI-based validation (Gemini)

## Goal

Demonstrate how automated QA pipelines can validate application behavior independently of production systems.
